# Apple App Store Submission & Rejection Handling

## Overview

Navigate App Store rejections systematically. Extract the specific rejection reason, implement code or metadata fixes, verify the build, and prepare for resubmission without guesswork.

## Workflow

### Step 1: Parse the Rejection Email

Read the rejection email carefully:

- **What section failed?** (functionality, metadata, privacy, content, etc.)
- **What is the exact violation?** (quote it)
- **What device/config triggered it?** (e.g., "MacBook Pro 14\" with notch")
- **What guideline references are cited?** (e.g., "4.2.1", "5.2.5")

Track rejections in `.rejections.md`.

### Step 2: Diagnose Root Cause

The review feedback is often the symptom, not the cause. Investigate:

**Code issues:**
- Does the app branch on device capabilities? Check if review device has that capability.
- Are required UI elements conditional? Code path untested on review hardware.
- Missing or wrong entitlements in Info.plist?

**Metadata issues:**
- App name, subtitle, or description violate guidelines?
- Screenshots stale or don't match current build?
- IAP product bundle ID mismatch?

### Step 3: Implement the Fix

**Code pattern:** Always instantiate required UI elements; conditionally show/hide based on device capabilities.

**Metadata pattern:** Update App Store Connect fields (name, subtitle, description, screenshots, IAP products) against specific guidelines.

### Step 4: Verify Before Resubmit

```bash
xcodebuild clean && xcodebuild build
plutil -p YourApp/Info.plist
codesign -d --entitlements - YourApp.app
```

### Step 5: Resubmit

1. Increment CFBundleVersion
2. Update release notes with fix description
3. Verify screenshots and metadata one more time
4. Build → Archive → Distribute via Xcode
5. Monitor submission queue in App Store Connect

## Common Rejections

| Rejection | Root Cause | Fix |
|-----------|-----------|-----|
| Can't quit app | Quit button only in one code branch | Add app menu with Quit ⌘Q; always instantiate both code paths |
| Subtitle duplicates name | Generic or redundant subtitle | Change to specific feature description in App Store Connect |
| IAP product not found | Bundle ID mismatch between code and App Store | Verify product ID in App Store Connect matches code exactly |
| Privacy policy missing | No URL provided in metadata | Add privacy policy URL to App Store Connect → App Information |

## Decision Tree

```
Rejection received
  ├─ Functionality issue?
  │  ├─ Code path unreachable → Unconditional instantiation + conditional visibility
  │  ├─ UI element missing → Add app menu or always-visible button
  │  └─ Entitlement wrong → Fix Info.plist and signing
  │
  ├─ Metadata issue?
  │  ├─ Name/subtitle generic → Update App Store Connect form
  │  ├─ Screenshots stale → Re-screenshot and upload
  │  └─ IAP product wrong → Verify bundle ID in App Store Connect
  │
  └─ Privacy/policy issue?
     ├─ Policy URL missing → Add to App Store Connect
     └─ Violates guidelines → Read specific rule; redesign/restrict feature
```
