# Claude Code Skill Budget Audit

## Overview

Claude Code allocates ~2% of context for skill descriptions. When this budget overflows:
- Skill descriptions are stripped (readability drops)
- Skill count hidden (no visibility into available skills)
- Performance degrades (fewer skills discoverable)

This skill provides a systematic audit, identification, and remediation workflow.

## Workflow

### Step 1: Detect the Problem

**Symptom:** Claude Code banner warning or notice in settings.json:

```
⚠️ Skill descriptions stripped — your skill cache exceeds the 2% budget
```

Or observe:
- Skill descriptions empty or truncated
- Skill count says "742 hidden"
- Some expected skills not showing

### Step 2: Quantify Current State

**Get plugin count:**

```bash
find ~/.claude/plugins -maxdepth 1 -type d | wc -l
# Output: 37 plugins (example)
```

**Estimate skill count:**

```bash
find ~/.claude/plugins -name "SKILL.md" -o -name "*.skill.yaml" | wc -l
# Output: ~575 skills
```

**Check current cost:**

```bash
du -sh ~/.claude/plugins
# Output: 12M (size of all skill manifests)

# Rough calculation:
# 2% of ~50K tokens = 1000 tokens budget
# Current: 575 skills × ~80 bytes each = 46KB ≈ 11.5K tokens
# Status: 1050% of budget — massive overage
```

### Step 3: Identify Heavy Bundles

Scan plugins for size and skill count:

```bash
#!/bin/bash
echo "Plugin Size Analysis:"
for dir in ~/.claude/plugins/*/; do
    plugin=$(basename "$dir")
    size=$(du -sh "$dir" 2>/dev/null | cut -f1)
    count=$(find "$dir" -name "SKILL.md" | wc -l)
    echo "$size | $count skills | $plugin"
done | sort -hr | head -15
```

### Step 4: Classify for Removal

For each heavy bundle, ask:

1. **Active usage?** Did I invoke this in past 30 days? (Check git log, ~/tasks/, recent projects)
   - Yes → Keep
   - No → Remove

2. **Niche/domain-specific?**
   - Financial-services bundle: Only active if banking/fintech work
   - Bitwize-music: Only if music production
   - cc-skills-golang: Only if Go development
   - Result: If not active in that domain → Remove

3. **Duplicate?**
   - Multiple superpowers installations
   - Multiple copies of same plugin
   - Result: Always remove duplicates

**Classification example:**

| Bundle | Size | Skills | Usage | Action |
|--------|------|--------|-------|--------|
| financial-services | 2.1M | 200 | 0 (no banking work) | Remove |
| bitwize-music | 1.8M | 52 | 0 (not musician) | Remove |

| superpowers (dup) | 0.6M | 28 | Duplicate | Remove |
| teifi-coding | 0.8M | 35 | 0 | Remove |

**Expected outcome:** Removing 4 bundles = ~6M saved, leaving ~6M, well under budget.

### Step 5: Create Phased Remediation Plan

Don't remove everything at once — test incrementally:

**Phase 1: Low-risk removals (unused, niche bundles)**
```bash
# Back up first
cp -r ~/.claude/plugins ~/.claude/plugins.backup

# Remove unused bundles
rm -rf ~/.claude/plugins/financial-services-bundle
rm -rf ~/.claude/plugins/bitwize-music
rm -rf ~/.claude/plugins/cc-skills-golang

# Restart Claude Code
# Expected: Budget warning gone, normal skill descriptions restored
```

**Phase 2: Remove duplicates**
```bash
rm -rf ~/.claude/plugins/superpowers-v2-duplicate
# or if there are multiple with same name:
find ~/.claude/plugins -name "superpowers" -type d | tail -n +2 | xargs rm -rf
```

**Phase 3: Validate (critical)**
```bash

# Should drop from 37 to ~15

# Check skill count
find ~/.claude/plugins -name "SKILL.md" | wc -l
# Should drop from 575 to ~150-200

# Restart Claude Code and verify:
```

## Acceptance Criteria

Audit complete when:

- [ ] Plugin count reduced to 10-20 (from 30+)
- [ ] Skill count reduced to 150-250 (from 500+)
- [ ] Budget warning gone (no stripped descriptions)
- [ ] Full skill descriptions visible in settings
- [ ] "Hidden skills" message cleared
- [ ] All expected skills still available (core bundles kept)
- [ ] No broken references (e.g., if a removed skill is used elsewhere)
- [ ] Remediation plan documented in ~/tasks/audit-log.md
