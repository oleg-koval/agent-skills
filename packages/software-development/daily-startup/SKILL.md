# Daily Startup

> 🤖 *Auto-generated by **weekly-pattern-learner** · obsidian-pr-sync and obsidian-task-rollover share the same vault config and morning-routine trigger and are consistently invoked together — unified here as a single morning entry point*

Run both Obsidian daily-note routines in sequence, in the correct order, with shared vault configuration. Carry yesterday's unfinished tasks forward first, then refresh the PR review queue for today.

## When to Use

- "Start my day" / "run morning routine" / "daily startup"
- Scheduled morning trigger (e.g., 08:00 weekdays)
- Remote CCR agent starting a morning session

## Configuration

- `VAULT_DAILY` — absolute path to the daily notes folder (e.g. `/Users/you/obsidian/vault/Lead/Daily`)
- `GH_USERNAME` — GitHub login for PR author filtering
- `ORG_PREFIX` — org name to group PRs as "Work" (e.g. `Teifi-Digital`)

Infer from context or existing vault files if not specified by the user.

## Workflow

### Step 1 — Task rollover (yesterday → today)

Identify yesterday's workday date and carry any unfinished tasks forward:

```bash
YESTERDAY=$(python3 -c "
from datetime import date, timedelta
d = date.today() - timedelta(days=1)
while d.weekday() >= 5:
    d -= timedelta(days=1)
print(d)
")
SOURCE="${VAULT_DAILY}/${YESTERDAY}.md"
```

Follow the full workflow in `packages/software-development/obsidian-task-rollover/SKILL.md`:
- Extract all `- [ ]` lines from the source note (excluding `## PRs to review` and already-migrated `[>]` lines)
- Replace them with `- [>] task → [[...TARGET_DATE]]` in the source
- Write them under `## Carried over` in today's note

Skip this step silently if the source note does not exist or has no incomplete tasks.

### Step 2 — PR sync (GitHub → today's note)

Fetch open PRs where the user is assigned or review-requested, and write a grouped `## PRs to review` section into today's note.

Follow the full workflow in `packages/software-development/obsidian-pr-sync/SKILL.md`:
- Run `gh search prs` for review-requested and assigned PRs
- Filter bots, drafts, self-authored
- Group by Work (`ORG_PREFIX`) vs. Personal
- Sort oldest-first within each group; flag PRs ≥ 7 days old with ⚠️
- Replace the section if it already exists (idempotent)

### Step 3 — Report

After both steps complete, print a single summary:

```
Morning startup complete — YYYY-MM-DD
  Task rollover: N tasks carried from YESTERDAY (or "nothing to carry")
  PR queue:      X work PRs, Y personal PRs
```

Surface any error (vault path missing, gh auth expired) immediately rather than silently skipping.

## Order matters

Task rollover must run before PR sync so `## Carried over` appears above `## PRs to review` in today's note, consistent with the standard daily template section order.

## Invocation patterns

**Manual:** "start my day" / "run morning routine" / "daily startup"

**Scheduled (08:00 weekdays):** Run both steps automatically. Print the one-line summary.

**Remote CCR agent:** Clone the vault repo, run both steps, commit and push. The Obsidian Git plugin pulls changes locally.
